# Out of Town Toggle Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a silent admin "Out of town" toggle that hides the Emergency Fix option from the client portal. Server enforces. No public indication the admin is away.

**Architecture:** Add `outOfTown` boolean column to `AdminSettings`. The `/api/availability` route stops returning `isAfterHours` and instead returns a coalesced `emergencyAvailable = isAfterHours && !outOfTown`. The `/api/portal/tickets` POST silently downgrades sneaky emergency requests when `outOfTown` is true. Admin gets a new toggle section on `/admin/account`. The client portal looks identical regardless of toggle state — emergency button just doesn't render after hours when on.

**Tech Stack:** Next.js (custom build per `AGENTS.md` — read `node_modules/next/dist/docs/` if API uncertain), Prisma 5.22, PostgreSQL (Supabase), React, Tailwind. No test runner; verification is manual + `npx tsc --noEmit` + `npm run lint`.

**Spec:** `docs/superpowers/specs/2026-05-08-out-of-town-toggle-design.md`

## Task 3: Replace `isAfterHours` with `emergencyAvailable` in `/api/availability`

This is the privacy-preserving rename. The raw `outOfTown` value never leaves the server.

**Files:**
- Modify: `app/api/availability/route.ts`

- [ ] **Step 1: Compute and return `emergencyAvailable`**

Open `app/api/availability/route.ts`. Replace the body of `GET()` so it loads `outOfTown` and computes `emergencyAvailable`. Specifically:

- After `const row = await prisma.adminSettings.findUnique(...)`, the existing `settings` object stays unchanged (it only feeds the pure `computeAvailability` / `isAfterHours` calls — adding `outOfTown` to it would force a `lib/availability.ts` change we want to avoid).
- After the existing `const afterHours = isAfterHours(settings, now);`, add:
  ```ts
  const outOfTown = row?.outOfTown ?? false;
  const emergencyAvailable = afterHours && !outOfTown;
  ```
- In the `NextResponse.json({...})` payload, replace `isAfterHours: afterHours,` with `emergencyAvailable,`. Keep everything else identical.

The full GET should end up as:

```ts
export async function GET() {
  const row = await prisma.adminSettings.findUnique({ where: { id: "global" } });

  const settings: AdminSettingsInput = {
    timezone: row?.timezone ?? "America/New_York",
    hours: (row?.hours as WeeklyHours | undefined) ?? DEFAULT_HOURS,
    oooEnabled: row?.oooEnabled ?? false,
    oooFrom: row?.oooFrom ?? null,
    oooUntil: row?.oooUntil ?? null,
    oooMessage: row?.oooMessage ?? null,
    holidays: row?.holidays ?? [],
  };

  const now = new Date();
  const availability = computeAvailability(settings, false, now);
  const afterHours = isAfterHours(settings, now);
  const outOfTown = row?.outOfTown ?? false;
  const emergencyAvailable = afterHours && !outOfTown;
  const emergencyFeeCents = row?.emergencyFeeCents ?? 5000;

  return NextResponse.json(
    {
      ...availability,
      settings: serializeSettings(settings),
      emergencyAvailable,
      emergencyFeeCents,
    },
    {
      headers: { "Cache-Control": "no-store" },
    },
  );
}
```

(`isAfterHours` is intentionally removed from the response payload.)
